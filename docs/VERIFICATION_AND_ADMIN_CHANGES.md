# Изменения: верификация и админка

## Как пройти верификацию (пошагово)

### Шаг 1. Заполнить Профиль (вкладка «Профиль»)
- **1. Основная информация:** Фамилия, Имя, Отчество, Дата рождения, Гражданство, Пол, Телефон
- **2. Паспорт:** Серия, номер, даты, кем выдан, загрузить скан
- **3. Водительское удостоверение:** Номер, категории, даты, скан
- Нажать **«Сохранить»**

### Шаг 2. (Опционально) Документы и гражданства
Можно заполнить **только форму профиля** — при «Отправить на верификацию» данные автоматически перенесутся в новые таблицы.

Или добавить вручную:
- **Документы** — вкладка «Документы» → «Добавить документ» (паспорт, ВУ)
- **Гражданства** — вкладка «Гражданства» → «Добавить гражданство»

### Шаг 3. Отправить на верификацию
- Во вкладке «Профиль» в блоке статуса нажать **«Отправить на верификацию»**
- Данные из формы (паспорт, ВУ, гражданство) автоматически перенесутся в новые таблицы

### Шаг 4. Админ верифицирует
- Админ заходит в `/admin/users` → выбирает водителя → кнопка «Верифицировать»

## Где «каша» (старое + новое)

| Что | Где | Куда сохраняется |
|-----|-----|------------------|
| ФИО, дата рождения, пол, телефон | DriverCardForm (вкладка Профиль) | `carrier_profiles` (legacy) |
| Паспорт (серия, номер, скан) | DriverCardForm → вкладка «2. Паспорт» | `carrier_profiles` (legacy) |
| ВУ (номер, категории) | DriverCardForm → вкладка «3. ВУ» | `carrier_profiles` (legacy) |
| Документы | DocumentsList (отдельная вкладка) | `driver_documents` (новая) |
| Гражданства | CitizenshipsList (отдельная вкладка) | `driver_citizenships` (новая) |

**Верификация проверяет:** `carrier_profiles` (ФИО, пол) + `driver_documents` (паспорт, ВУ) + `driver_citizenships` (гражданство).

При нажатии «Отправить на верификацию» данные из формы (паспорт, ВУ, гражданство) автоматически копируются в новые таблицы, если их там ещё нет.

## Что изменилось у админа (frontend)

### Страница пользователя `/admin/users/:id`

1. **Теги статусов верификации:**
   - Верифицирован (зелёный)
   - Ожидает проверки (жёлтый) — для `submitted`, `waiting_verification`
   - Запрос (синий)
   - Отклонён (красный)
   - Приостановлен (красный)
   - Не верифицирован (серый)

2. **Кнопки действий:**
   - **Верифицировать** — когда статус `submitted` или `waiting_verification`
   - **Отклонить** — с обязательной причиной (модальное окно)
   - **Приостановить** — для верифицированных водителей, с причиной
   - **Восстановить** — для приостановленных

3. **API:** используется `/admin/drivers/:userId/verify`, `reject`, `suspend`, `restore` вместо `/admin/users/:id/verify`.

## Как работает новая система верификации

### Для водителя

1. **Заполнение карточки** — `/cabinet/driver/card`:
   - Вкладки: Профиль, Документы, Гражданства, Контакты, Запросы на изменение
   - Заполняет обязательные поля (ФИО, дата рождения, пол, паспорт, ВУ, гражданство)

2. **Отправка на верификацию** — кнопка «Отправить на верификацию» в `ProfileVerificationStatus`

3. **Статусы:**
   - `not_submitted` / `draft` / `not_verified` — можно редактировать
   - `submitted` / `waiting_verification` — ожидает проверки админом
   - `verified` — верифицирован, может работать
   - `rejected` — отклонён, можно отправить повторно
   - `suspended` — приостановлен админом
   - `request` — есть запрос на изменение (одобренный админом)

4. **После верификации** — для изменения паспорта/ВУ/ФИО нужен «Запрос на изменение» (вкладка «Запросы на изменение»).

### Для админа

1. Список водителей — фильтр по статусу верификации
2. Карточка водителя — кнопки Верифицировать / Отклонить / Приостановить / Восстановить
3. Отклонение и приостановка — обязательная причина в модальном окне

## Вход водителя в кабинет

- После логина водитель перенаправляется на `/cabinet/driver`
- С главной (`/`) — через «Кабинет» в меню пользователя
- Прямой переход: `http://localhost:3000/cabinet/driver` или `/cabinet` (редирект по роли)
