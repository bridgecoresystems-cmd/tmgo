## 2. Поля формы прицепа (из таблицы ТЗ)

| № | Поле | UI компонент | Обяз. | Условие показа |
|---|------|-------------|-------|----------------|
| 1 | VIN номер | `n-input` маска 17 симв. | да | всегда |
| 2 | Марка | `n-select` каскадный (vehicle_makes) | да | всегда |
| 3 | Модель | `n-select` зависит от марки | да | всегда |
| 4 | Год выпуска | `n-input-number` 1950–текущий | опц. | всегда |
| 5 | Номер шасси | `n-input` | опц. | всегда |
| 6 | Цвет | `n-select` (VEHICLE_COLORS) | да | всегда |
| 7 | Госномер | `n-input` | да | всегда |
| 8 | Год выпуска (тех. состояние) | — | опц. | не дублировать, убрать |
| 9 | Грузоподъёмность | `n-input-number` суффикс «т» | да | всегда |
| 10 | Объём кузова (м³) | `n-input-number` суффикс «м³» | опц. | всегда |
| 11 | **Тип кузова** | `n-select` (TRAILER_BODY_TYPES) | да | всегда |
| 12 | **ADR (опасные грузы)** | `n-checkbox-group` (ADR_CLASSES) | опц. | всегда |
| 13 | GPS/ГЛОНАСС | `n-switch` | опц. | всегда |
| 14 | Страхование | номер полиса + `n-date-picker` | опц. | всегда |
| 15 | Тип топлива | `n-select` | **да** | только если bodyType ∈ REEFER_BODY_TYPES |
| 16 | Объём топливного бака (л) | `n-input-number` | **да** | только если bodyType ∈ REEFER_BODY_TYPES |
| 17 | Тип КПП | `n-select` | **да** | только если bodyType ∈ REEFER_BODY_TYPES |
| 18 | Расход топлива в час (л/ч) | `n-input-number` | **да** | только если bodyType ∈ REEFER_BODY_TYPES |

**Дополнительные поля (не в таблице, но нужны):**

| Поле | UI | Обяз. | Условие |
|------|-----|-------|---------|
| Тип прицепа (полу/прицеп) | `n-radio-group`: semi / drawbar | да | всегда |
| Количество осей | `n-input-number` 1–5 | опц. | всегда |
| Количество паллетомест | `n-input-number` | опц. | curtainsider/mega/reefer |
| Диапазон температур мин. (°C) | `n-input-number` | опц. | только reefer/multi_temp |
| Диапазон температур макс. (°C) | `n-input-number` | опц. | только reefer/multi_temp |
| Владение прицепом | `n-radio-group`: own / company / leased | опц. | всегда |

### 3.2 Новая таблица `driver_trailers`

Создать миграцию `packages/backend/drizzle/0019_driver_trailers.sql`:

```sql
DO $$ BEGIN
  CREATE TYPE "trailer_type" AS ENUM ('semi', 'drawbar');
EXCEPTION WHEN duplicate_object THEN NULL; END $$;

DO $$ BEGIN
  CREATE TYPE "trailer_body_type" AS ENUM (
    'curtainsider','mega','double_deck',
    'reefer','multi_temp','isothermal',
    'flatbed','lowboy','extendable','coil_well',
    'tipper_rear','tipper_side','tipper_3way','walking_floor',
    'tanker_fuel','tanker_food','tanker_chemical','tanker_gas','silo',
    'container_20','container_40','container_swap',
    'timber','livestock','car_carrier','dumper_tipping','oversized'
  );
EXCEPTION WHEN duplicate_object THEN NULL; END $$;

CREATE TABLE IF NOT EXISTS "driver_trailers" (
  -- Идентификация
  "id"                      uuid PRIMARY KEY DEFAULT gen_random_uuid() NOT NULL,
  "carrier_id"              uuid NOT NULL REFERENCES "carrier_profiles"("id") ON DELETE CASCADE,

  -- Основные
  "vin"                     varchar(17),
  "trailer_type"            "trailer_type" NOT NULL DEFAULT 'semi',
  "body_type"               "trailer_body_type" NOT NULL,
  "plate_number"            text NOT NULL,
  "color"                   text NOT NULL,

  -- Марка / модель (справочник или кастом)
  "make_id"                 uuid REFERENCES "vehicle_makes"("id"),
  "model_id"                uuid REFERENCES "vehicle_models"("id"),
  "custom_make"             text,
  "custom_model"            text,

  -- Характеристики
  "year"                    smallint,
  "chassis_number"          text,
  "capacity_tons"           numeric(6,2) NOT NULL,
  "volume_m3"               numeric(7,2),
  "axle_count"              smallint,
  "pallet_places"           smallint,

  -- ADR (массив классов: ['3', '6.1', ...])
  "adr_classes"             text[] DEFAULT '{}',

  -- Температурный режим (только для reefer/multi_temp)
  "temp_min_c"              smallint,
  "temp_max_c"              smallint,

  -- GPS / страховка
  "has_gps"                 boolean DEFAULT false,
  "insurance_policy_num"    text,
  "insurance_expires_at"    timestamp,

  -- Холодильный агрегат (только для reefer/multi_temp)
  "ref_fuel_type"           "fuel_type",
  "ref_fuel_tank_l"         integer,
  "ref_transmission"        "transmission_type",
  "ref_fuel_consumption_ph" numeric(5,1),

  -- Владение
  "ownership"               "vehicle_ownership",

  -- Жизненный цикл
  "is_active"               boolean DEFAULT true NOT NULL,
  "created_at"              timestamp DEFAULT now() NOT NULL,
  "updated_at"              timestamp DEFAULT now() NOT NULL
);

CREATE INDEX IF NOT EXISTS "idx_driver_trailers_carrier" ON "driver_trailers"("carrier_id");
CREATE INDEX IF NOT EXISTS "idx_driver_trailers_body_type" ON "driver_trailers"("body_type");
CREATE INDEX IF NOT EXISTS "idx_driver_trailers_active" ON "driver_trailers"("is_active");
```

> **Важно:** enum'ы `fuel_type`, `transmission_type`, `vehicle_ownership` уже созданы в миграции тягача `0018_*`. Не пересоздавать.

### 3.3 Таблица `vehicle_couplings` (история сцепок)

Добавить в ту же миграцию `0019_driver_trailers.sql`:

```sql
CREATE TABLE IF NOT EXISTS "vehicle_couplings" (
  "id"            uuid PRIMARY KEY DEFAULT gen_random_uuid() NOT NULL,
  "carrier_id"    uuid NOT NULL REFERENCES "carrier_profiles"("id") ON DELETE CASCADE,
  "tractor_id"    uuid NOT NULL REFERENCES "driver_vehicles"("id"),
  "trailer_id"    uuid NOT NULL REFERENCES "driver_trailers"("id"),
  "coupled_at"    timestamp DEFAULT now() NOT NULL,
  "decoupled_at"  timestamp,         -- NULL = сцепка активна прямо сейчас
  "notes"         text,
  "is_active"     boolean DEFAULT true NOT NULL,
  "created_at"    timestamp DEFAULT now() NOT NULL
);

CREATE INDEX IF NOT EXISTS "idx_couplings_tractor"  ON "vehicle_couplings"("tractor_id");
CREATE INDEX IF NOT EXISTS "idx_couplings_trailer"  ON "vehicle_couplings"("trailer_id");
CREATE INDEX IF NOT EXISTS "idx_couplings_active"   ON "vehicle_couplings"("is_active");

-- Constraint: у одного тягача не может быть двух активных сцепок
CREATE UNIQUE INDEX IF NOT EXISTS "idx_couplings_tractor_active"
  ON "vehicle_couplings"("tractor_id") WHERE "is_active" = true;

-- Constraint: у одного прицепа не может быть двух активных сцепок
CREATE UNIQUE INDEX IF NOT EXISTS "idx_couplings_trailer_active"
  ON "vehicle_couplings"("trailer_id") WHERE "is_active" = true;
```

## 4. Backend API (Elysia.js)

Создать новый файл `packages/backend/src/routes/cabinet/driver-trailers.ts`.

Структура аналогична `driver-vehicles.ts` — derive для авторизации, те же паттерны.

### 4.1 `GET /cabinet/driver/trailers`
Список прицепов водителя (`is_active = true`).  
Response: `{ id, plateNumber, bodyType, trailerType, brand, model, capacityTons, isActive }[]`

### 4.2 `POST /cabinet/driver/trailers`
Создать прицеп. Валидация через `t.Object()`:

```typescript
body: t.Object({
  vin:                  t.Optional(t.RegExp(/^[A-HJ-NPR-Z0-9]{17}$/i)),
  trailerType:          t.Union([t.Literal('semi'), t.Literal('drawbar')]),
  bodyType:             t.Union(trailerBodyTypeIds.map(id => t.Literal(id)) as any),
  plateNumber:          t.String({ minLength: 2, maxLength: 20 }),
  color:                t.String({ minLength: 1 }),
  makeId:               t.Optional(t.String()),
  modelId:              t.Optional(t.String()),
  customMake:           t.Optional(t.String()),
  customModel:          t.Optional(t.String()),
  year:                 t.Optional(t.Number({ minimum: 1950, maximum: new Date().getFullYear() })),
  chassisNumber:        t.Optional(t.String()),
  capacityTons:         t.Number({ minimum: 0.1, maximum: 200 }),
  volumeM3:             t.Optional(t.Number({ minimum: 0 })),
  axleCount:            t.Optional(t.Number({ minimum: 1, maximum: 10 })),
  palletPlaces:         t.Optional(t.Number({ minimum: 0 })),
  adrClasses:           t.Optional(t.Array(t.String())),
  tempMinC:             t.Optional(t.Number({ minimum: -40, maximum: 30 })),
  tempMaxC:             t.Optional(t.Number({ minimum: -40, maximum: 30 })),
  hasGps:               t.Optional(t.Boolean()),
  insurancePolicyNum:   t.Optional(t.String()),
  insuranceExpiresAt:   t.Optional(t.String()),   // ISO date string
  refFuelType:          t.Optional(t.String()),
  refFuelTankL:         t.Optional(t.Number()),
  refTransmission:      t.Optional(t.String()),
  refFuelConsumptionPh: t.Optional(t.Number()),
  ownership:            t.Optional(t.Union([t.Literal('own'), t.Literal('company'), t.Literal('leased')])),
})
```

**Серверная валидация:** если `bodyType` ∈ `REEFER_BODY_TYPES` → проверить что `refFuelType`, `refFuelTankL`, `refTransmission`, `refFuelConsumptionPh` указаны.

### 4.3 `GET /cabinet/driver/trailers/:id`
Полная карточка прицепа. Проверить что `carrier_id` принадлежит текущему водителю.

### 4.4 `PATCH /cabinet/driver/trailers/:id`
Обновить поля. Все поля optional. Та же серверная валидация рефки.

### 4.5 `DELETE /cabinet/driver/trailers/:id`
Soft delete: `is_active = false, updated_at = now()`.  
**Дополнительно:** если у прицепа есть активная сцепка — автоматически завершить её (`decoupled_at = now(), is_active = false`).

## 5. API сцепок (Couplings)

Добавить в тот же файл `driver-trailers.ts` или создать отдельный `driver-couplings.ts`.

### 5.1 `GET /cabinet/driver/couplings`
Активные сцепки водителя.  
Response: 
```typescript
{
  id: string,
  tractor: { id, plateNumber, brand, model },
  trailer: { id, plateNumber, bodyType, brand, model },
  coupledAt: string,
}[]
```

### 5.2 `POST /cabinet/driver/couplings`
Создать новую сцепку.

```typescript
body: t.Object({
  tractorId: t.String(),
  trailerId: t.String(),
  notes:     t.Optional(t.String()),
})
```

**Серверная логика:**
1. Проверить что `tractorId` и `trailerId` принадлежат текущему водителю
2. Проверить что тягач не имеет активной сцепки → если есть: вернуть ошибку `409 Conflict` с сообщением `"Тягач уже прицеплен к другому прицепу"`
3. Проверить что прицеп не имеет активной сцепки → аналогично
4. Создать запись в `vehicle_couplings`

### 5.3 `DELETE /cabinet/driver/couplings/:id`
Завершить сцепку: `decoupled_at = now(), is_active = false`.  
Проверить что `carrier_id` сцепки принадлежит текущему водителю.

import { REEFER_BODY_TYPES } from '@tmgo/shared'

const isReefer = computed(() =>
  form.bodyType !== null && REEFER_BODY_TYPES.includes(form.bodyType as any)
)
const isMultiTemp = computed(() => form.bodyType === 'multi_temp' || form.bodyType === 'reefer')

// Сброс рефрижераторных полей при смене типа кузова
watch(() => form.bodyType, (newType) => {
  if (!REEFER_BODY_TYPES.includes(newType as any)) {
    form.refFuelType = null
    form.refFuelTankL = null
    form.refTransmission = null
    form.refFuelConsumptionPh = null

})

## 11. Важные правила

- **Soft delete везде** — `is_active = false`, физически не удалять
- **Уникальная активная сцепка** — один тягач / один прицеп = максимум одна активная сцепка (enforced через partial unique index в БД)
- **При деактивации прицепа** — автоматически завершать активную сцепку (в API)
- **При деактивации тягача** — аналогично завершать активную сцепку (добавить в `driver-vehicles.ts`)
- **REEFER_BODY_TYPES** — импортировать из `@tmgo/shared`, не хардкодить на фронте
- **Валидация рефки** — и на фронте (computed `isReefer`) и на бэке (в POST/PATCH handler)
- **Авторизация** — `carrier_id` всегда из профиля, не из body запроса
- **Snapshot для рейсов** — в будущем `trip` будет хранить `tractor_id + trailer_id + coupling_id` — архитектура уже готова
